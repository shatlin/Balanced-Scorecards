GridView & DataFormatString in post beta 2

Vinny Davi asked a good question on ASP.NET Forums. Vinny had GridView with BoundField which was setup to format date field with DataFormatString and problem was that string formatting just didn't work. It had worked with beta 2, and after upgrading to Release Candidate it stopped working.

Answer is simple and is explained thoroughly here. Shortly, there's now html encoding applied by default for the data before it is formatted. Therefore one needs to set HtmlEncode="false" to the BoundField so that formatting has expected effect.

Just note that html encoding is there to protect from script injection.

Joe,

if you get error that BoundColumn does not have HtmlEncode property, you are probably mixing that with DataGrid? DataGrid has BoundColumn and it does *not* have HtmlEncode property while GridView has BoundField which does have HtmlEncode property.

So far I've seen most of the problems being solved with the knowledge of setting HtmlEncode to false or by reading this piece of documentation.

BoundField.dataFormatString
http://msdn2.microsoft.com/en-us/library/system.web.ui.webcontrols.boundfield.dataformatstring.aspx

If your case is not solved by these, feel free to send me email about it with more details what you are doing.
