    <asp:BoundField DataField="gsLastUpdDateTime" DataFormatString="{0:d t}" HeaderText="Last Updated" />

    I've tried umpteen variations of the DataFormatString and nothing yet has altered the output in the GridView.  I can tell you that if I take the "0:" out of the string, I get a format exception error.

    When the Visual Studio "Edit Fields" diaglog is open and the BoundField properties are displayed, the value {0:d t} is in the DataFormatString property.  The "of little actual help" text that is displayed when the cursor is in the property shows "The formatting that is applied to the bound value.  For example, "{0:d}" or "{0:c}"

    OK, so I try just {0:d}, but it has no effect on the output either.

    Of course, when you press F1 for context sensitive help, you get get help for Fields Dialog Box.  So, you scroll down a little and click on Bound Field (right?),  then you scroll down a little more until you see DataFormatString in the Remarks section and click on that.

    Yeah, so now you think you're getting somewhere:

    BoundField.DataFormatString Property
    Note: This property is new in the .NET Framework version 2.0.   
    Gets or sets the string that specifies the display format for the value of the field.

    So, you click on the "Example" link and you see just one example (and not of a date, or course):
    dataformatstring="{0:F4}%"

    Kind of makes you think that this property is only for numeric values, not dates.  Although I've seen vague references in other places that it can also be used for formatting date values... but no working examples.

    How can this be so difficult?  There must be something very simple that I am overlooking?

    Make me feel like saying "forget bound controls"... they'll just be obsolete in the next release anyway, right?
